github.io域名指向

发表于 2016-03-29

###如何配置自己的gihub.io的域名指向。
进入用hexo生成的本地my_blog_name文件夹

#进入我的博客目录
cd my_blog_name

#进入source文件夹
cd source

#新建CNAME文件
vi CNAME

编辑CNAME文件 在CNAME文件里插入你要指向的域名，注意域名不要带http www等
例如我的CNAME文件内容就是 cocoa1024.com

域名注册

以下是几个域名服务提供商，随便选一个点进去去注册就好了

  • 万网
  • 万网
  • 万网
  • 万网

####tips:
尽量不要选择.cn域名，因为需要备案，如果个人用户可以选择注册最近比较火的.me域名，一看后缀就知道很适合做个人网站，比如著名的iOS开发者Mattt大神的个人网站就是mattt.me

.me域名里有很多有趣的域名都被注册了比如：about.me,kiss.me,还有fuck(和谐).me，五个字符以内的短域名已经基本都被抢注了，如果做iOS开发者的技术博客的话，建议可以用如下关键字去注册，cocoa，framewrok，iosDev，applefun等，例如我的博客域名就是cocoa1024.com顺便致敬一下你懂的那个社区
当然你也可以给你的偶像致敬，你爱的人，或者几分戏谑的挑逗一下巨头们，比如你可以叫hardsoft，去对抗微软什么的。当然你也可以很贱的叫localhost8080（这个域名已经被注册了）.这些都没人管，只要你喜欢，当然你还得有点￥RMB或者$dollar。如果你没钱，乖乖的选个便宜的吧。

DNS解析

去你的域名提供商的主页或者dnspod去解析域名吧。
打开对应的后台，添加你的域名，并且给你的域名添加两条A记录，一条@，一条www 这样保证无论是通过www.yourdomain.com或者直接访问yourdomain.com的方式都可以直接访问到你的页面。

####tips:
如果你不知道你的github.io的ip，那么打开终端，ping yourname.github.io
control+c结束，把获取到的ip填到A记录对应字段里就OK了

做完以上步骤之后，你只需要等待，域名解析服务会把对你域名解析推到全球，一般会是10分钟-72小时生效不等。这样你通过你的域名就可以访问你自己的主页了。听起来是不是很兴奋呢？！

基于github.io和hexo搭建技术博客

发表于 2016-03-28

github配置

github提供了pages功能。能够生成属于你自己的站点。
登录自己的github，新建一个仓库，仓库的名字必须和你的github用户名一致

生成好自己的仓库以后，clone到本地

git clone https://github.com/username/username.github.io
进入仓库生成index.html 
#进入你的仓库
cd username.github.io
#生成index.html
echo "Hello World" > index.html
提交并推送到远端 
#add 到本地仓库
git add --all
#提交到本地仓库
git commit -m "Initial commit"
#推送到远程仓库
git push -u origin master

这时候去访问http://your_username.github.io 你的站点主页生成好了。虽然只有一个hello world。

Hexo

有了站点，可我不会js怎么办。没关系有Hexo。hexo会为你自动生成博客目录，你只需要关注其中的博客内容就可以了。类似的工具还有octopress，wordpress等。这里选用Hexo因为相比其他几种工具配置相对容易。目录结构清晰。
安装Hexo需要Node.js环境 所以先去官网下载安装包，pkg文件。
安装完成之后，安装hexo。

#安装hexo工具
npm install -g hexo

Hexo的配置与使用

先生成一个博客

#生成博客
hexo init my_blog_name

hexo会自动为你创建一个名为my_blog_name的文件夹，里面就包含你的博客框架。
进入my_blog_name/source/_posts里面会有一个默认的hellowrold.md。这个就是hexo为你生成的博客内容，用markdown编辑器打开 随便编辑一下。
然后在本地启动server，预览

#生成博客
hexo generate
启动本地服务器
hexo server

这时在浏览器里输入localhost:4000就可以看到你的博客页面了，hexo已经为你加载好了默认的landscape主题，很漂亮吧，很有成就感吧。

Hexo关联github

进入刚刚生成的my_blog_name文件夹 打开_config.yml。这是hexo的配置文件，这里包括了你的基础信息等等，暂时不关心。直接翻到最下边找到deploy:并编辑

deploy:
  type: git
  repository: git@github.com:SecondDog/SecondDog.github.io.git
  branch: master

注意编辑的时候一定要注意半角的冒号和他后边的空格，不然会导致没法解析。
编辑保存之后进入my_blog_name文件夹，

hexo deploy

这是你的hexo就会自动生成的my_logo_name项目推送到你的
https://github.com/username/username.github.io仓库中，去浏览器里输入访问http://your_username.github.io。就会发现Hexo项目已经和github.io产生了关联。
